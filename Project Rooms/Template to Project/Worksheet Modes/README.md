# Worksheet Mode Rules

Use this folder to keep project-management spreadsheet redesign and rollout rules separated by worksheet mode.

Before doing workbook work, identify the active worksheet mode and read that mode's rule file. After completing work in a mode, write any reusable lesson to that same mode file before marking the work complete. Cross-cutting connector, rollback, and general rollout rules may remain in `Project Spreadsheet Expense Placement Rules.md`, but worksheet-specific rules should live here.

For every worksheet-mode migration, assume the agent does not already know the full change set. Before rolling a template workbook or template worksheet to other projects, compare the approved template against each target's current worksheet and build a mode-specific change map. Propagate all intentional template changes for that mode, including formulas, comments/notes, controls, formatting, named ranges, helper cells, table structure, and dependent references. Do not migrate only the one cell or formula that triggered the request unless Wes explicitly limits the rollout to that single item.
