# Spreadsheet Change Request Workflow

When Wes is reviewing or designing a spreadsheet, he may give a series of change requests before he is ready for implementation.

Rules:

- Treat these requests as a pending change list, not approval to edit the workbook immediately.
- Do not implement the pending spreadsheet changes until Wes explicitly asks to implement them.
- Before making any spreadsheet file change, ask Wes to close the workbook first.
- Preserve any formatting Wes has already made unless he explicitly asks for formatting changes.
- When implementation begins, make a backup copy before editing the workbook.
- After editing, verify the workbook opens cleanly in Excel.

# Project Management Workbook Edit Workflow

When making changes to a project management spreadsheet stored in a Teams/SharePoint synced folder:

- First determine whether the original project workbook is closed.
- If the workbook is open, ask Wes to close it before making changes.
- If the workbook is closed, copy the workbook into the appropriate project room and make the edits on the project-room copy.
- After edits are complete, verify the workbook opens cleanly in Excel.
- Copy the verified edited workbook back to the original Teams/SharePoint folder where that project workbook belongs.
- Do not leave the project-room copy as the only updated version unless Wes explicitly asks for a test-only copy.
