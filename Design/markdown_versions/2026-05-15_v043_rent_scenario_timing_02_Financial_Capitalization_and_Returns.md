# Capitalization Structure

## Capital Source Rule

The project is now modeled on the basis that all design, development, build, and delivery capital is raised from Class A and Class B members.

The model does not assume a development loan or construction loan. The only new loan contemplated in the workbook is a stabilization refinance/takeout loan after the project reaches the refinance point.

## Class A Beneficiary Structure

Class A participants are anticipated to participate as preferred capital beneficiaries of the trust structure.

### Class A Preferred Return

* 8% cumulative preferred annual return
* Accruing from date of contribution
* Calculated pro rata based on staged capital calls

Preferred returns are anticipated to accrue throughout:

* entitlement,
* development,
* construction,
* and stabilization.

After Class A preferred return, Class A capital return, and Class B capital return priorities have been satisfied, Class A also participates in residual distributions through the Class A/Class B residual pool.

---

## Class A Distribution Priority

Upon stabilization and refinancing, proceeds are anticipated to flow generally in the following order:

1. Pay existing first mortgage debt
2. Pay existing private debt principal and any then-due private debt interest obligations
3. Pay accrued Class A preferred return
4. Return Class A contributed capital
5. Return/pay Class B capital from remaining proceeds and operating/sale cash flow according to the Class B terms
6. Residual distributions according to the adjustable Sponsor/Class pool split

---

## Class B Beneficiary Structure

Class B participants are anticipated to participate as subordinate participating beneficiaries within the trust structure.

Class B capitalization is modeled as the completion-capital source required to fully fund the project after Class A capital. The working capitalization case assumes sponsor-controlled post-frame construction, including sponsor-owned equipment and sponsor-controlled crew capacity.

In the current workbook, Class B is calculated as:

* selected total project cost,
* less Class A capital.

Based on the current selected sponsor-controlled low project cost case of **900,000** and Class A capital of **300,000**, modeled Class B capitalization is **600,000**. If the selected case changes in the workbook, Class B capitalization updates automatically.

The selected project cost now includes a specific **150,000** sponsor property payment. The payment is itemized in the Cost Analysis bridge and shown as a Year 0 Sponsor distribution in the Waterfall.

Class B participation may include:

* residual project cash flow participation,
* long-term appreciation participation,
* refinance participation,
* and post-stabilization operating distributions.

After capital return priorities, residual distributions are currently modeled as a Sponsor/Class pool split:

* Sponsor residual share: **50.00%**
* Class A and Class B combined residual pool: **50.00%**

The Sponsor residual share is editable in the workbook. Class A and Class B split the remaining non-sponsor residual pool in proportion to their contributed capital.

Based on the current Class A and Class B capital amounts:

| Participant | Residual Share |
|---|---:|
| Sponsor | 50.00% |

The Class A/Class B split is calculated from current capital amounts:

Within the 50.00% Class pool, Class A receives one-third of the pool and Class B receives two-thirds of the pool.

# Annual Cash Roll-Forward

The Annual Model now includes a cash roll-forward instead of treating each year's cash as isolated.

The Annual Model shows:

* current-year cash before refinance/sale,
* net refinance or sale proceeds,
* cash available before distributions,
* waterfall distributions,
* ending cash balance,
* loan balance,
* and property/refinance value.

Cash available before distributions is calculated as:

```text
Prior-year ending cash balance
+ current-year cash before refinance/sale
+ net refinance/sale proceeds
```

Waterfall distributions are then pulled from the Waterfall tab. Ending cash balance is calculated after those distributions.

Under the current waterfall settings, available cash is distributed each year, so the modeled ending cash balance is currently **0** in the early operating years. The structure is now in place to carry retained cash forward if the waterfall or reserve assumptions later leave cash undistributed.

The refinance loan is modeled as the lesser of:

1. **70% of stabilized value**, or
2. **total existing debt plus Class A and Class B capitalization**.

Based on the current workbook values:

* stabilized value: approximately **1,064,090**,
* 70% of stabilized value: approximately **744,863**,
* total existing debt plus Class A and Class B capitalization: **1,094,000**,
* modeled refinance loan amount: approximately **744,863**,
* net refinance proceeds after existing debt payoff: approximately **550,863**,
* indicative amount remaining after existing debt and Class A capital: approximately **250,863**.

The refinance loan is not modeled as a source of development or construction capital. It is modeled as a takeout loan after stabilization, with net proceeds flowing through the waterfall: Class A first, then Class B, then residual distributions.
